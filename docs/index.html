🧭 架構總覽

FearW 是 Hermés Agent 的 Fork，專為 CPA（CLIProxyAPI） 路由架構設計。所有 AI 模型流量都經過 CPA 代理：

你的指令 → Hermés Agent (FearW) → CPA (8080) → OpenAI / OpenRouter / MiniMax / ...

CPA 同時充當 管理介面，可以在一個介面裡管理多個 API Key 和多個 Provider。

📦 需要安裝什麼

  1. CPA — CLIProxyAPI（路由、管理）
  2. FearW — Hermés Agent 本體
  3. VOICEVOX（可選）— 日語 TTS

⚡ 快速前提

  • macOS（已測試）
  • Python 3.11+
  • Git
  • API Key（不提供，需自行申請）

常見問題

Q: CPA 和 FearW 是分開的嗎？

是的。CPA 是一個獨立的 OpenAI-compatible proxy 伺服器（預設 port 8080），FearW 把它當作上遊 AI API。兩者独立運行，CPA 重啟不影響 FearW，反之亦然。

Q: 現有的 Hermés Agent（NousResearch 原版）還能用嗎？

可以。兩者可以同時運行，使用不同 port 和 Profile，不會衝突。FearW 只走 CPA；原版 Hermés 可以直接連 Provider。

Q: 我的 API Key 放在哪裡？

放在 CPA 的認證目錄（~/.cli-proxy-api/），FearW 完全不接觸原始 Key。CPA 管理介面（:8080/management）可以安全地新增、刪除、測試 Key。
