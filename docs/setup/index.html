FearW 安裝

FearW 是 Hermés Agent 的 Fork，專為 CPA 架構設計。它本身不連接 AI Provider，所有流量都經過 CPA。

1. Clone 倉庫

下載 FearW

git clone https://github.com/whypuss/hermes-agent-fearw.git ~/.hermes/hermes-agent-fearw

2. 建立虛擬環境

1

建立 venv

cd ~/.hermes/hermes-agent-fearw python3 -m venv .venv
2

啟動 venv（每次操作前都要執行）

source .venv/bin/activate
3

安裝依賴

pip install -e .

這會安裝 FearW 到 venv，並安裝所有必需的依賴。

3. Profile 初始化

創建 Profile

hermes init --profile fearw

這會在 ~/.hermes/ 創建一個名為 fearw 的 Profile 配置目錄。

配置 Model（CPA 路由）

編輯 ~/.hermes/profiles/fearw/config.yaml：

model: default: "MiniMax-M2.7" # 預設模型 provider: "cliproxyapi" base_url: "http://127.0.0.1:8080/v1" # CPA API Key（用來連接本地 CPA） providers: cliproxyapi: api_key: "fearw123" # 你的 CPA 密碼 dashboard: port: 9120 # 避開原版 Hermés 端口 password: "fearw123" cpa_api_proxy: true public: true # 允許外部訪問（可選）

4. 同時運行原版 Hermés 和 FearW

兩者可以同時運行，端口互不衝突：

# 原版 Hermés hermes agent run --port 8765 # FearW cd ~/.hermes/hermes-agent-fearw && source .venv/bin/activate hermes -p fearw agent run

5. Dashboard 模式（可選）

啟動 FearW Dashboard

cd ~/.hermes/hermes-agent-fearw && source .venv/bin/activate hermes -p fearw dashboard --port 9120 --public --password fearw123 --cpa-api-proxy --no-open
  • --port 9120：Dashboard 端口（避開 8765/9119）
  • --public：允許外部網絡訪問
  • --cpa-api-proxy：開啟 CPA 代理功能
  • --no-open：不自動打開瀏覽器
# 訪問 Dashboard http://localhost:9120

6. 驗證安裝

Doctor 檢查

cd ~/.hermes/hermes-agent-fearw && source .venv/bin/activate hermes -p fearw doctor

測試 CPA 連接

curl http://127.0.0.1:8080/v1/models \ -H "Authorization: Bearer 你的CPA密碼"

正常情況下返回可用模型列表。

7. 升級 FearW

cd ~/.hermes/hermes-agent-fearw && git pull
