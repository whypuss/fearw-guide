CPA 安裝與配置

CPA（CLIProxyAPI）是 FearW 的核心——所有 AI 流量都經過它。它是一個獨立的 OpenAI-compatible proxy 伺服器，支援多個 Provider、多個 API Key、管理介面、流量監控。

1. 安裝 CLIProxyAPI

1

確認 Python 環境

python3 --version # 需要 3.9+
2

使用 pip 安裝

pip install cli-proxy-api # 或 pip3 install cli-proxy-api

或者用独立二进制（推薦）：

brew install cli-proxy-api # macOS
3

驗證安裝

cli-proxy-api --version

2. 配置文件

config.yaml 位置

CPA 會在以下順序查找配置：

  1. ./config.yaml（當前目錄）
  2. ~/.cli-proxy-api/config.yaml
  3. /etc/cli-proxy-api/config.yaml

建議放在 ~/.cli-proxy-api/config.yaml

最小配置示例

host: "127.0.0.1" port: 8080 api-keys: - "fearw123" # CPA 本地訪問密碼 debug: false # OpenRouter + FreeRouter（推薦，完全免費） openai-compatibility: - name: "openrouter" base-url: "https://openrouter.ai/api/v1" api-key-entries: - api-key: "sk-or-v1-你的OpenRouterKey" models: - name: "openrouter/free" alias: "free"

真實 API Key 建議員放在 auth-dir 目錄（見下方），不要直接寫在 config.yaml。

快速開始：OpenRouter + FreeRouter（推薦）

OpenRouter 提供每月 $1 免費 Credit，配合 openrouter/free 模型，可以完全免費使用。

  1. 到 openrouter.ai/keys 申請 API Key
  2. 在 CPA config 加入 OpenRouter provider（如下方完整示例）
  3. FearW 設定 model: free 即可

3. API Key 管理（安全方式）

auth-dir 目錄

預設 ~/.cli-proxy-api/。在 config.yaml 指定：

auth-dir: "~/.cli-proxy-api"

認證文件格式

在 auth-dir 裡，每個 Provider 建一個目錄，例如：

~/.cli-proxy-api/openrouter/ keys/ key_001.txt # 內容：sk-or-v1-xxxxx key_002.txt # 內容：sk-or-v1-yyyyy

或者用 JSON 格式（可在管理介面編輯）：

~/.cli-proxy-api/openrouter/keys.json { "keys": [ { "key": "sk-or-v1-xxxxx", "label": "我的 OpenRouter Key" } ] }

4. 啟動 CPA

前台運行（調試用）

cli-proxy-api --config ~/.cli-proxy-api/config.yaml

後台運行（生産用）

nohup cli-proxy-api --config ~/.cli-proxy-api/config.yaml \ >> ~/.cli-proxy-api/cpa.log 2>&1 & # 或使用 launchd / systemd 管理

5. 管理介面

本地訪問

http://127.0.0.1:8080/management

認證方式：HTTP Basic Auth，用 config.yaml 裡的任意 api-key。

功能概覽

  • 新增 / 刪除 API Key
  • 選擇模型（Model Picker）
  • 測試 API 連線
  • 流量監控
  • 上 / 下游配置

6. Provider 設定

Providerbase-url說明
OpenRouterhttps://openrouter.ai/api/v1包含 FreeRouter（免費模型）
OpenAIhttps://api.openai.com/v1官方
自定義任意任何 OpenAI-compatible 端點

FreeRouter（免費模型）

OpenRouter 提供 openrouter/free 模型，會自動路由到最便宜的免費模型。使用方式：

# 在 CPA config 加入 models: - name: "openrouter/free" alias: "free" # 在 FearW 或任何客戶端使用 model: free

7. 測試 CPA 是否正常

API 端點測試

curl http://127.0.0.1:8080/

正常返回：

{"endpoints":["POST /v1/chat/completions","POST /v1/completions","GET /v1/models"],"message":"CLI Proxy API Server"}

模型列表測試

curl http://127.0.0.1:8080/v1/models \ -H "Authorization: Bearer 你的CPA密碼"
