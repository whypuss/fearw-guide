進階設定

1. 使用非原版端口

為什麼需要非原版端口？

原版 Hermés Agent 默認使用 port 8765。FearW 和原版同時運行時，必須使用不同端口。

指定端口的方法

# 方法一：啟動時指定端口 hermes -p fearw gateway run --port 9120 # 方法二：在 config.yaml 設定 gateway: port: 9120 host: "0.0.0.0" # 允許外部訪問

Dashboard 端口

hermes -p fearw dashboard --port 9121 --password fearw123

2. 終端命令 /fearw（Shell Alias）

macOS Zsh — 添加 Alias

編輯 ~/.zshrc：

echo 'alias fearw="cd ~/.hermes/hermes-agent-fearw && source .venv/bin/activate && hermes -p fearw"' >> ~/.zshrc source ~/.zshrc

之後在終端直接輸入 fearw 即可激活 FearW。

用 fearw 快速啟動各模式

# 激活後 fearw agent run # 啟動 Agent fearw gateway run # 啟動 Gateway fearw dashboard # 啟動 Dashboard fearw chat # 啟動 Chat 模式

3. 多 Profile 管理

創建新 Profile

hermes init --profile myprofile

Profile 目錄結構

~/.hermes/ profiles/ fearw/ config.yaml # FearW 配置 myprofile/ config.yaml # 另一個配置

切換 Profile

hermes -p myprofile agent run

4. FreeRouter（OpenRouter 免費模型）

什麼是 FreeRouter？

openrouter/free 是 OpenRouter 提供的特殊模型，會自動將請求路由到當前最便宜的免費或 deep-discount 模型。

實測會路由到 minimax/minimax-m2.5:free 等模型。

在 CPA config 加入

openai-compatibility: - name: "openrouter" base-url: "https://openrouter.ai/api/v1" api-key-entries: - api-key: "sk-or-v1-你的OpenRouterKey" models: - name: "openrouter/free" alias: "free"

在 FearW 使用

model: default: "free"

7. 常見問題排查

CPA 連接失敗

  1. 確認 CPA 正在運行：ps aux | grep cli-proxy
  2. 確認端口：lsof -i :8080
  3. 測試本地：curl http://127.0.0.1:8080/

Telegram Bot 沒反應

  1. 檢查 Bot Token 是否正確
  2. 確認 Gateway 已啟動並監聽
  3. 檢查 admin_ids 是否包含你的 User ID
  4. 查看日誌確認 Telegram 平台已加載

模型返回錯誤

  1. 檢查 CPA 的 API Key 是否有效
  2. 確認 CPA config 裡的模型名稱正確
  3. 在 CPA 管理介面測試該模型
